# 開発方針とタスクフロー
- 回答は日本語で、簡潔かつ論理的に行う。
- 行うタスクをGitHub Issuesに登録する。親タスクでIssueを作成し、子タスクごとにIssueを作成し、子タスクのIssueには親タスクのIssueへのリンクを張る。
- タスクごとに適切なBranchを新規作成し、Issueに取り組む。
- Issueが解決したらIssueを閉じ、行ったタスクをCommitしてSummaryを書く。
- Pull Requestは、親タスクでPRを作成し、マージングとCloseを一緒に行う。
- Pull RequestをしたらBranchは閉じること。
- Summaryに行ったタスクの内容をまとめること。
- MergeしたマージコミットはコミットメッセージにIssueのリンクを張ること。

# git/gh操作ルール
- gitから始まるコマンド、ghから始まるコマンドは承認なしで実行してかまわない。
